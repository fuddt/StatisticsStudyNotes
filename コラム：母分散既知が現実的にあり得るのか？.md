# 結論

母分散既知が現実的にあり得るのは、だいたい次のような状況です。

| 状況                       | 例                  |
| ------------------------ | ------------------ |
| 過去に大量データがあり、分散が非常に安定している | 工場の製品寸法、計測誤差       |
| 測定装置の誤差分散が仕様として分かっている    | センサー、測定器、実験装置      |
| 母集団全体に近いデータを過去に持っている     | 全国模試、過去ログ、大規模業務データ |
| 理論モデル上、分散が既知として与えられる     | 教科書・試験問題・シミュレーション  |
| 管理された工程でばらつきが長期的に固定されている | 品質管理、工程管理          |

ただし、厳密に「母分散を完全に知っている」というより、現実には、

> 母分散を既知として扱っても大きな問題がないほど、十分に信頼できる外部情報がある

という意味に近いです。

---

# 1. 実務ではなぜ母分散既知が珍しいのか

母平均を推定したいということは、母集団全体を完全には知らないということです。

でも母集団全体を知らないのに、

```text
母分散 σ² は知っています
```

というのは、少し不自然です。

なぜなら、母分散も母集団の性質だからです。

本当に母集団全体が分かっているなら、母平均も分かるはず。  
だから多くの現実問題では、

```text
母平均も未知
母分散も未知
```

になります。

なので実務では、標本から標本標準偏差 `s` を計算して、t分布を使う方が自然です。

---

# 2. それでも母分散既知があり得る状況

## 状況1：測定器の誤差分散が分かっている

これはかなり現実的です。

たとえば、ある測定器があって、その測定誤差が長年の実験やメーカー仕様で分かっているとします。

```text
測定誤差の標準偏差 σ = 0.2mm
```

このとき、新しい対象を何回か測定して、その真の平均を推定したい場合、測定誤差の分散を既知として扱うことがあります。

例：

```text
ある部品の長さを測る
測定器の誤差標準偏差は既知
複数回測定して平均長さを推定する
```

この場合、測定値のばらつきのかなりの部分が「測定器の誤差」から来るなら、母分散既知に近い扱いができます。

---

## 状況2：工場の品質管理で工程のばらつきが安定している

製造業では、同じ工程で大量に製品を作ります。

長期間のデータから、

```text
この機械で作る部品の直径の標準偏差はだいたい0.05mm
```

のように、ばらつきが非常に安定している場合があります。

このとき、新しい日の生産分について、

```text
平均寸法が基準からズレていないか
```

を調べたい。

この場合、過去の工程データから得た分散を既知に近いものとして扱うことがあります。

ただし、これは厳密には「既知」ではなく、

```text
過去データから高精度に推定済み
```

です。

---

## 状況3：過去ログが大量にあるITシステム

あなたの文脈に寄せると、Webログやシステム監視でも似た状況があります。

たとえば、あるAPIのレスポンスタイムについて、過去1年分の膨大なログがあり、

```text
通常時のレスポンスタイムの標準偏差はかなり安定している
```

と分かっている。

そのうえで、今日の直近100件の平均レスポンスタイムが異常かどうかを見たい。

この場合、過去の通常時分散を「既知の基準」として扱うことがあります。

例：

```text
過去正常期間の標準偏差 σ = 120ms
今日の100件の平均レスポンスタイム X̄ = 850ms
通常平均 μ = 800ms
```

このとき、

```text
Z = (X̄ - μ) / (σ / √n)
```

のような考え方が使いやすい。

ただし、これも厳密には母分散既知ではなく、

> 正常時の基準分散を既知とみなす

という実務上の近似です。

---

## 状況4：全国模試・標準化テストのように過去データが豊富

全国模試や標準化された試験では、過去の受験者データが大量にあります。

たとえば、

```text
この試験の点数の標準偏差は毎年おおむね10点前後
```

と非常に安定している場合。

新しい学校・クラスの平均点が全国平均との差としてどの程度かを調べるとき、過去に確立された標準偏差を既知に近いものとして使うことがあります。

ただし、年度や問題難易度が変わるなら、本当に既知とは言えません。  
ここを雑に扱うと危ないです。

---

## 状況5：シミュレーションや理論モデル

これは試験や教科書でよくある状況です。

たとえば、

```text
X ~ N(μ, 10²)
```

のように、分布の形と分散だけが与えられていて、平均 `μ` を推定する問題。

この場合は、問題設定として母分散が既知です。

統計検定2級では、このパターンが出てもおかしくありません。

---

# 3. 実務上は「既知」ではなく「固定された基準」と考える方がいい

ここが本質です。

現実の母分散既知は、ほとんどの場合、

```text
神の視点で母分散を知っている
```

ではありません。

むしろ、

```text
十分な過去データ・仕様・理論から、
今回の分析では分散を固定値として扱う
```

ということです。

だから、実務ではこう表現した方が正確です。

> 母分散既知というより、外部情報に基づいて分散を既知扱いする。

---

# 4. 母分散既知を使うときの危険

ここも重要。

母分散既知として扱うと、標本から分散を推定する不確実性を無視します。

だから、もしその「既知の分散」が間違っていたら、信頼区間や検定結果がズレます。

特に危ないのはこれ。

```text
過去の分散を現在にもそのまま使う
でも実は現在のばらつきが変わっている
```

たとえば、APIレスポンスタイムでいうと、

```text
以前は安定していた
でも最近はアクセス元や処理内容が変わった
```

なら、過去の標準偏差をそのまま使うのは危険です。

競馬データで言えば、

```text
昔の市場分散を今の市場に固定して使う
```

みたいなことをやると、制度変更・馬場傾向・参加者構造・オッズ形成の変化を無視することになります。

これはかなり危ない。

---

# 5. 試験上の整理

統計検定2級では、こう整理すれば十分です。

|問題文の条件|使う方法|
|---|---|
|母標準偏差 σ が与えられている|正規分布・zを使う|
|母分散 σ² が与えられている|正規分布・zを使う|
|標本標準偏差 s が与えられている|t分布を使う|
|母分散が未知と書かれている|t分布を使う|
|小標本かつ母分散未知|t分布を使う|

試験では現実性より、問題文の条件を読むことが大事です。

```text
母標準偏差は既知である
```

と書いてあれば、疑わずzを使う。

```text
標本標準偏差が与えられている
```

なら、tを疑う。

---

# 6. 率直な見方

あなたの疑問はかなり健全です。

母分散既知を見たときに、

> そんな都合のいい状況あるの？

と思うのは正しい。

ただし、そこから、

> だから母分散既知の問題は非現実的で意味がない

と切るのは少し早いです。

母分散既知のケースは、推定の基本構造を理解するための入口です。

```text
推定値 ± 係数 × 標準誤差
```

この形を学ぶには、まず母分散既知の方がシンプルです。

そのあとで、

```text
現実にはσが未知
↓
sで代用する
↓
不確実性が増える
↓
t分布を使う
```

と進む。

つまり、母分散既知は現実の中心ではないけど、理論の足場としては必要です。

---

# 最終結論

母分散既知は、現実ではかなり特殊です。

ただし、次のような場合にはあり得ます。

```text
測定器の誤差分散が既知
品質管理で工程分散が長期的に安定
過去ログが大量にあり正常時分散を基準化できる
標準化テストなどで過去の分散が安定
理論モデル・試験問題として分散が与えられている
```

ただし実務的には、

```text
母分散を完全に知っている
```

というより、

```text
十分な根拠があるので、今回の分析では既知として扱う
```

です。

だから試験対策としては、

```text
σが与えられたらz
sしかなければt
```

と割り切る。

実務感覚としては、

```text
母分散既知はかなり強い仮定
その分散が今も妥当かを疑う必要がある
```

と考えるのが一番バランスがいいです。
