# 2. 実験計画とは何か

実験計画とは、簡単に言えば、

```text
条件の違いによる効果を、できるだけ正しく測るための実験の設計
```

です。

たとえば、3種類の教材A・B・Cの効果を比べたいとします。

```text
教材Aを使った生徒
教材Bを使った生徒
教材Cを使った生徒
```

のテスト点数を比較する。

このとき、ただ適当に比べると危険です。

なぜなら、教材の効果以外にも、点数に影響するものがあるからです。

```text
もともとの学力
勉強時間
先生の教え方
クラスの雰囲気
家庭環境
体調
```

こういう要因が混ざります。

実験計画は、こうした余計な要因の影響をできるだけ抑えて、

```text
本当に教材の違いによる差なのか？
```

を見やすくするための設計です。

---

# 3. 処理とは何か

実験で比較したい条件のことを、**処理**といいます。

たとえば、

|実験|処理|
|---|---|
|教材の効果比較|教材A、教材B、教材C|
|肥料の効果比較|肥料なし、肥料少量、肥料多量|
|薬の効果比較|偽薬、低用量、高用量|
|UI改善の比較|旧UI、新UI|
|広告効果の比較|広告なし、広告A、広告B|

つまり、処理とは、

```text
結果に影響するか調べたい条件
```

です。

---

# 4. 処理群と対照群

実験では、よく次の2つを区別します。

|用語|意味|
|---|---|
|処理群|効果を調べたい処理を受ける群|
|対照群|比較の基準になる群|

たとえば、新薬の効果を調べるなら、

```text
処理群：新薬を飲む
対照群：偽薬を飲む
```

です。

教材の効果なら、

```text
処理群：新教材を使う
対照群：従来教材を使う
```

です。

重要なのは、処理群だけを見ても効果は分からないことです。

```text
新薬を飲んだ人の70%が改善した
```

だけでは不十分です。

なぜなら、

```text
何もしなくても自然に改善したのでは？
偽薬でも改善したのでは？
もともと軽症者が多かったのでは？
```

という可能性があるからです。

だから対照群が必要です。

---

# 5. 悪い実験例

次の実験を考えます。

```text
成績を上げる新教材を試したい。
希望者だけに新教材を使わせた。
その後、点数が上がった。
```

これを見て、

```text
新教材には効果がある
```

と言ってよいでしょうか。

かなり危険です。

なぜなら、希望者はもともとやる気が高い可能性があるからです。

```text
新教材を希望した人
= もともと勉強意欲が高い人
```

かもしれません。

この場合、点数が上がった理由は教材ではなく、もともとの意欲かもしれない。

つまり、処理群と対照群が比較可能になっていません。

---

# 6. フィッシャーの3原則

ここで出てくるのが、**フィッシャーの3原則**です。

フィッシャーの3原則は、実験計画の基本です。

```text
1. 無作為化
2. 反復
3. 局所管理
```

それぞれ意味があります。

|原則|目的|
|---|---|
|無作為化|偏りを減らす|
|反復|偶然のブレを見積もる|
|局所管理|余計なばらつきを減らす|

この3つは、試験でもかなり出やすいです。

---

# 7. 原則1：無作為化

無作為化とは、処理をランダムに割り付けることです。

たとえば、90人の生徒を3つの教材に分けます。

```text
30人：教材A
30人：教材B
30人：教材C
```

このとき、誰がどの教材を使うかをランダムに決めます。

```text
生徒90人
↓ ランダム割付
教材A群・教材B群・教材C群
```

これが無作為化です。

---

# 8. なぜ無作為化が必要なのか

無作為化の目的は、

```text
処理群同士を、できるだけ公平にすること
```

です。

もし先生が勝手に、

```text
成績上位者には教材A
成績中位者には教材B
成績下位者には教材C
```

と割り付けたら、教材の効果を比較できません。

結果として教材A群の点数が高くても、

```text
教材Aが良かったから
```

なのか、

```text
もともと成績上位者が多かったから
```

なのか分かりません。

無作為化は、こういう偏りを減らすためにあります。

---

# 9. 無作為抽出と無作為化の違い

前回も触れましたが、ここはもう一度整理します。

|用語|何をランダムにするか|目的|
|---|---|---|
|無作為抽出|母集団から標本を選ぶ|標本を母集団の代表に近づける|
|無作為化・無作為割付|対象を処理群に分ける|処理群同士を比較可能にする|

かなり重要です。

```text
無作為抽出：誰を調査対象にするか
無作為割付：誰にどの処理を与えるか
```

です。

この2つを混同すると、実験計画の理解が崩れます。

---

# 10. 原則2：反復

反復とは、同じ処理を複数の対象に行うことです。

たとえば、教材Aを1人だけに使わせるのではなく、30人に使わせる。

```text
教材A：30人
教材B：30人
教材C：30人
```

このように、各処理を複数回観測することが反復です。

---

# 11. なぜ反復が必要なのか

反復が必要なのは、偶然のブレを見積もるためです。

たとえば、教材Aを使った1人が90点を取ったとします。

でも、それだけでは教材Aが良いとは言えません。

その1人がたまたま優秀だっただけかもしれません。

逆に、教材Bを使った1人が50点だったとしても、たまたま体調が悪かっただけかもしれません。

だから、複数人で試します。

```text
1人だけ → 偶然かもしれない
複数人 → 平均やばらつきを見られる
```

反復によって、処理ごとの平均だけでなく、処理内のばらつきも見られるようになります。

この「処理内のばらつき」が、分散分析の分母側につながります。

---

# 12. 反復がないと分散分析できない

かなり大事です。

分散分析では、

```text
群間のばらつき
群内のばらつき
```

を比べます。

でも、各群に1つずつしかデータがなかったら、群内のばらつきが分かりません。

たとえば、

|教材|点数|
|---|--:|
|A|80|
|B|70|
|C|60|

これだけだと、

```text
Aが一番高い
```

とは言えます。

でも、

```text
A群の中でどれくらいバラつくのか
```

が分かりません。

だから、偶然のブレか処理の効果かを判断しにくい。

反復は、統計的判断のために必要です。

---

# 13. 原則3：局所管理

局所管理とは、結果に影響しそうな条件をできるだけそろえることです。

または、条件が似たもの同士で比較することです。

たとえば、農業実験で肥料A・B・Cを比較するとします。

畑の場所によって、日当たりや土の状態が違うかもしれません。

もし肥料Aを日当たりの良い場所にだけ使い、肥料Bを日当たりの悪い場所にだけ使ったら、肥料の効果なのか日当たりの違いなのか分かりません。

そこで、似た条件の場所ごとにまとめて、その中で肥料を比較します。

これが局所管理の発想です。

---

# 14. 局所管理の例

教材実験なら、

```text
学力レベルが近い生徒同士で比較する
同じ先生が教える
同じ授業時間にする
同じ試験問題にする
```

農業実験なら、

```text
日当たりが似た区画内で比較する
土壌条件が似た区画内で比較する
水やり条件をそろえる
```

WebのA/Bテストなら、

```text
同じ期間で比較する
同じ流入経路で比較する
デバイス種別をそろえる、または層別する
```

という発想です。

局所管理は、

```text
余計なばらつきを減らす
```

ためにあります。

---

# 15. ブロック化とは何か

局所管理の代表的な方法が、**ブロック化**です。

ブロック化とは、似た条件の対象をグループに分け、そのグループ内で処理を比較する方法です。

たとえば、生徒のもともとの学力が結果に影響しそうなら、

```text
高学力ブロック
中学力ブロック
低学力ブロック
```

に分けます。

そして、それぞれのブロック内で教材A・B・Cを割り付けます。

```text
高学力：A, B, C
中学力：A, B, C
低学力：A, B, C
```

このようにすると、学力の違いによるばらつきをある程度コントロールできます。

---

# 16. ブロック化の直感

ブロック化なしで教材を比較すると、こうなるかもしれません。

|教材|生徒の傾向|
|---|---|
|A|成績上位者が多い|
|B|成績中位者が多い|
|C|成績下位者が多い|

これでは、教材の効果が分かりません。

ブロック化すると、

|学力ブロック|教材A|教材B|教材C|
|---|---|---|---|
|高学力|あり|あり|あり|
|中学力|あり|あり|あり|
|低学力|あり|あり|あり|

となります。

こうすれば、各学力レベルの中で教材を比較できます。

つまり、ブロック化は、

```text
邪魔な要因をそろえて、公平な比較に近づける
```

方法です。

---

# 17. 層化抽出とブロック化の違い

ここも混同しやすいです。

|用語|場面|目的|
|---|---|---|
|層化抽出|標本調査|母集団を代表する標本を取る|
|ブロック化|実験計画|余計なばらつきを抑えて処理効果を見やすくする|

似ています。

どちらもグループに分けます。

でも目的が違います。

```text
層化抽出：標本をうまく取るため
ブロック化：実験をうまく比較するため
```

です。

---

# 18. 一元配置実験とは何か

ここから分散分析に接続します。

一元配置実験とは、**1つの要因について、複数の水準を比較する実験**です。

たとえば、

```text
要因：教材
水準：教材A、教材B、教材C
```

です。

または、

```text
要因：肥料
水準：肥料なし、肥料少量、肥料多量
```

です。

「一元」とは、見ている要因が1つという意味です。

---

# 19. 要因と水準

ここは用語として重要です。

|用語|意味|例|
|---|---|---|
|要因|結果に影響するか調べたい条件の種類|教材、肥料、薬|
|水準|要因の具体的な値・種類|教材A・B・C|

例：

|実験|要因|水準|
|---|---|---|
|教材比較|教材|A, B, C|
|肥料比較|肥料量|0g, 10g, 20g|
|薬の比較|投与量|なし、低用量、高用量|

つまり、

```text
要因 = 何を変えるか
水準 = どう変えるか
```

です。

---

# 20. 一元配置分散分析とは何か

一元配置分散分析とは、

```text
1つの要因の水準によって、平均に差があるかを調べる方法
```

です。

たとえば、教材A・B・Cの平均点を比較します。

|教材|平均点|
|---|--:|
|A|78|
|B|72|
|C|68|

このとき、

```text
A, B, Cの平均点に、本当に差があると言えるのか？
それとも、たまたま標本で差が出ただけなのか？
```

を調べるのが一元配置分散分析です。

---

# 21. なぜt検定を何回もやらないのか

3群以上の平均を比べるなら、

```text
A vs B
A vs C
B vs C
```

と、2標本t検定を何回もやればよさそうに見えます。

でも、これは危険です。

理由は、第1種の過誤が増えるからです。

1回の検定で有意水準5%なら、間違って差があると言ってしまう確率は5%です。

でも、検定を何回もすると、どこかで偶然有意になりやすくなります。

だから、3群以上の平均比較では、まず分散分析を使います。

```text
3群以上の平均差を見る入口 = 分散分析
```

です。

---

# 22. 分散分析の基本発想

分散分析の発想は、かなり直感的です。

```text
群の平均同士が大きく違う
```

だけでは不十分です。

なぜなら、データ自体のばらつきが大きければ、平均が少し違っても偶然かもしれないからです。

そこで、分散分析では次を比べます。

```text
群間のばらつき：群の平均同士がどれくらい違うか
群内のばらつき：同じ群の中でどれくらいバラつくか
```

そして、

```text
群間のばらつきが、群内のばらつきに比べて十分大きいか
```

を見ます。

---

# 23. F比とは何か

分散分析で出てくるF比は、ざっくり言えばこれです。

```text
F比 = 群間のばらつき / 群内のばらつき
```

より正確には、平均平方を使いますが、最初はこの理解で十分です。

|値|解釈|
|---|---|
|F比が大きい|群間差が群内ばらつきに比べて大きい|
|F比が1に近い|群間差と群内ばらつきが同じくらい|
|F比が小さい|群間差が目立たない|

つまり、F比が大きいほど、

```text
群の平均に差がありそう
```

と考えます。

---

# 24. 例：群間差が大きく、群内ばらつきが小さい

教材A・B・Cの点数が次のようだったとします。

|教材A|教材B|教材C|
|--:|--:|--:|
|79|70|60|
|80|71|61|
|81|72|62|

各群の中では点数がかなりまとまっています。

平均はだいたい、

```text
A：80
B：71
C：61
```

です。

この場合、

```text
群内ばらつきは小さい
群間差は大きい
```

ので、教材による差がありそうです。

F比は大きくなりやすいです。

---

# 25. 例：群間差はあるが、群内ばらつきも大きい

次はどうでしょう。

|教材A|教材B|教材C|
|--:|--:|--:|
|60|50|40|
|80|75|70|
|100|90|85|

平均だけ見ると、Aが高そうです。

でも、各群の中のばらつきも大きいです。

この場合、

```text
群の平均差が、偶然のばらつきに埋もれている可能性
```

があります。

つまり、平均だけ見て判断するのは危険です。

分散分析は、このような状況で、

```text
群間差が群内ばらつきに比べて十分大きいか
```

を見ます。

---

# 26. 一元配置分散分析の帰無仮説

一元配置分散分析の帰無仮説は、

```text
すべての群の母平均は等しい
```

です。

教材A・B・Cなら、

```text
H0：μA = μB = μC
```

です。

対立仮説は、

```text
少なくとも1つの群の母平均が異なる
```

です。

ここが重要です。

対立仮説は、

```text
全部の平均が違う
```

ではありません。

```text
少なくともどこか1つに差がある
```

です。

---

# 27. 分散分析で有意だったら何が分かるか

分散分析で有意だった場合、分かるのは、

```text
少なくとも1つの群の平均が他と違いそう
```

ということです。

しかし、

```text
どの群とどの群が違うか
```

までは、分散分析だけでは分かりません。

たとえば、A・B・Cで有意差があっても、

```text
AとBが違うのか
AとCが違うのか
BとCが違うのか
```

は別途調べる必要があります。

これは多重比較の話になります。

統計検定2級では深くやりすぎなくてよいですが、

```text
ANOVA有意 = どこかに差がある
```

と押さえてください。

---

# 28. 実験計画と分散分析の接続

ここまでをつなげます。

実験計画では、

```text
処理の違いによる効果を見たい
```

わけです。

一元配置実験では、

```text
1つの要因の複数水準を比較する
```

わけです。

分散分析では、

```text
水準ごとの平均に差があるかを検定する
```

わけです。

つまり、

```text
実験計画でデータを作る
↓
一元配置実験として整理する
↓
分散分析で平均差を検定する
```

という流れです。

---

# 29. 競馬AIでの注意

ここは率直に言います。

競馬データ分析は、ほとんどの場合、実験ではありません。

観察研究です。

だから、たとえば、

```text
騎手Aの回収率が高い
馬場状態Bで戦略Xが強い
人気帯Cでモデルの期待値が高い
```

という結果があっても、それは実験で処理を割り付けたわけではありません。

```text
騎手をランダムに割り付けた
馬場状態をランダムに決めた
オッズをランダムに形成した
```

わけではない。

だから、因果としてはかなり弱い。

言えるのは、

```text
過去データ上、その条件で関連が観察された
```

までです。

これを超えて、

```text
この条件が利益の原因である
```

と言い切ると危険です。

ただし、実験計画の考え方は競馬AIにも応用できます。

たとえば、

```text
検証期間を事前に固定する
条件を後出しで選ばない
人気帯・競馬場・距離で層別する
NoBetを含めて評価する
同じルールを将来データに適用する
```

これは実験計画の精神に近いです。

あなたの研究でいうなら、

```text
後から勝った切り口を拾う
```

のは最悪に近いです。

それは実験ではなく、ノイズの採集です。

# 31. 今回のまとめ

|用語|意味|
|---|---|
|実験計画|効果を正しく測るための実験設計|
|処理|効果を調べたい条件|
|処理群|処理を受ける群|
|対照群|比較の基準になる群|
|無作為化|処理をランダムに割り付けること|
|反復|同じ処理を複数の対象に行うこと|
|局所管理|条件をそろえて余計なばらつきを減らすこと|
|ブロック化|似た条件ごとに分け、その中で比較すること|
|一元配置実験|1つの要因の複数水準を比較する実験|
|F比|群間のばらつき / 群内のばらつき|

```text
群間差が、群内ばらつきに比べてどれくらい大きいか
```

# 34. 次回につながる話

次回は、

```text
第35回：時系列：成長率・指数化・移動平均
```

```text
データの種類
```

次回は、時間の順番を持つデータ、つまり**時系列データ**を扱います。

たとえば、

```text
月ごとの売上
日ごとのアクセス数
```

```text

指数化
移動平均
トレンド
```

を見る必要があります。
